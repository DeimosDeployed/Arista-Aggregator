## 🚀 درباره پروژه

**AristaPanel** یک ابزار تماماً خودکار است که:
- از **بیش از ۵۰۰ کانال تلگرامی** و **۱۰+ سورس گیت‌هاب** کانفیگ استخراج می‌کند.
- کانفیگ‌ها را **اعتبارسنجی** کرده و تگ اختصاصی `ARISTA` یا `T.ME: @aristapanel` به آن‌ها اضافه می‌کند.
- **حذف خودکار کانال‌های مرده** (عدم آپدیت بیش از ۲ روز) و **بلاک موقت ۷ روزه** برای کانال‌های بدون پست جدید.
- **کش هوشمند** برای سورس‌هایی که ۲۴ ساعت متوالی فیلتر شده‌اند (هر ۶ ساعت یکبار اجرا می‌شود).
- **دسته‌بندی کانفیگ‌ها** به انواع: `vmess`, `vless`, `trojan`, `ss`, `hysteria2`, `hysteria`, `tuic`, `wireguard`
- **خروجی Tiered** با اندازه‌های ۵۰, ۱۰۰, ۱۵۰, ۲۰۰, ۲۵۰, ۳۰۰, ۴۰۰, ۵۰۰ و ALL همراه با **همپوشانی ۱۰ تایی** بین تیِرها.
- **ادغام خودکار** کانفیگ‌های تلگرام و گیت‌هاب در پوشه `configs/combined/`
- **حذف دایپلیکیت** بر اساس هش MD5.

> 🧠 تمام فرآیند هر **۶ ساعت یکبار** توسط GitHub Actions اجرا می‌شود و خروجی به مخزن پوش داده می‌شود.

## 📁 ساختار خروجی

پس از هر اجرا، فایل‌های زیر در پوشه `configs/` ایجاد می‌شوند:

```

configs/
├── telegram/           # کانفیگ‌های استخراج شده از تلگرام
│   ├── vmess.txt
│   ├── vless.txt
│   ├── all.txt
│   └── [هر دسته]/[tier].txt
├── github/             # کانفیگ‌های استخراج شده از گیت‌هاب
│   ├── vmess.txt
│   └── ...
└── combined/           # ادغام تلگرام + گیت‌هاب
├── vmess.txt
├── all.txt
└── [دسته‌بندی]/[tier].txt

```

خروجی **تییربندی شده** در هر دسته به صورت زیر است:
- `50.txt`, `100.txt`, `150.txt`, ..., `ALL.txt`
- هر تییر شامل `همپوشانی ۱۰ تایی` با تییر قبلی (برای پایداری اتصال)

## 🛠 نحوه عملکرد فنی

### 1. استخراج از تلگرام (`telegram_extractor.py`)
- اسکرپ کانال‌های عمومی با فرمت `https://t.me/s/...`
- استخراج آخرین پست‌ها (حداکثر ۱۵ کانفیگ بر کانال)
- بررسی `last post time` و حذف کانال‌های بدون پست جدید به مدت ۲ روز
- بلاک موقت ۷ روزه برای کانال‌های غیرفعال
- کش دائم برای کانال‌هایی که ۳ بار متوالی فیلتر شده‌اند (۲۴ ساعت عدم تلاش مجدد)

### 2. استخراج از گیت‌هاب (`github_extractor.py`)
- دریافت از ۱۰+ سورس مختلف (raw فایل‌های متنی)
- پشتیبانی از لینک‌های `cdn.jsdelivr.net` و `gitverse.ru`
- اعتبارسنجی و استانداردسازی لینک‌های `ss://`

### 3. ترکیب و تییربندی (`combine_configs.py`)
- حذف دایپلیکیت در سطح تمام سورس‌ها
- تولید فایل‌های `combined` با ذکر تعداد سورس تلگرام و گیت‌هاب
- ساختار Tiered با همپوشانی

## 🤖 خودکارسازی با GitHub Actions

فایل `.github/workflows/main.yml`:
- اجرای خودکار هر ۶ ساعت (`*/6 * * * *`)
- قابلیت اجرای دستی (`workflow_dispatch`)
- اجرا روی `ubuntu-latest` با تایم‌اوت ۴۵ دقیقه
- تأخیر تصادفی بین ۶۰ تا ۳۶۰ ثانیه برای جلوگیری از ریت لیمیت
- کامیت و پوش خودکار در صورت وجود تغییر
